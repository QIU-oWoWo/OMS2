# OMS 配件订单管理平台 前端交互 PRD v3.0

> **文档用途**：供代码 Agent 生成 OMS 前端可交互页面


---

## 1. 产品定位与设计原则

| 维度 | 说明 |
|------|------|
| 产品定位 | 配件供销体系"后端中枢"，负责订单全生命周期管理、异常调度、数据聚合与经营分析 |
| 目标用户 | 内部运营人员、仓管人员、客服主管、管理层 |
| 设计范式 | 混合范式：顶部状态 Tab（参考 17TRACK 全生命周期视图）+ 底部高密度数据表（参考 Shopify 后台） |
| 录入原则 | 精简录入，不暴露复杂表单；批量操作优先于逐条操作 |
| 信息密度 | 后台管理级密度，单屏展示 20-50 行数据，支持列自定义 |

| Tab 名称 | 对应状态码 | 说明 |
|----------|-----------|------|
| 待审核 | PENDING_REVIEW | 新创建待人工审核的订单 |

| 开票时间 | invoiceTime | DateTime | — |

#### 4.6.2 交互规则

| 场景 | 触发条件 | 系统行为 |
|------|---------|---------|

## 附录 A：状态机定义

### 订单状态流转

```
PENDING_REVIEW → SCHEDULING → PICKING → READY_TO_SHIP → IN_TRANSIT → DELIVERED → COMPLETED
       ↓              ↓           ↓            ↓              ↓           ↓
   EXCEPTION ←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←
```

- 任意状态均可转入 EXCEPTION
- EXCEPTION 处理完毕后可恢复至原流程节点或关闭订单
- COMPLETED 为终态，不可逆

### 预约单状态流转

```
PENDING_CONFIRM → CONFIRMED → EXECUTED（自动转正）
       ↓               ↓
   CANCELLED       EXPIRED（超期未执行）
```

### 定制订单状态流转

```
DRAFT → TECH_REVIEW → QUOTE_PENDING → IN_PRODUCTION → COMPLETED
  ↓          ↓              ↓
CANCELLED  REJECTED     REJECTED
```

### 400免费订单状态流转

```
PENDING_APPROVAL → APPROVED → SHIPPED
       ↓
   REJECTED
```

### 异常状态流转

```
PENDING → PROCESSING → RESOLVED → CLOSED
                ↓
           （升级）→ PENDING（重新分配处理人）
```
